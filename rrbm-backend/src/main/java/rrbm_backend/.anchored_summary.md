## Goal
Implement the Import Review System — a two-pass batch import flow where users manually verify every order and expense via a review modal before commit, with commit results persisted for historical tracking.

## Constraints & Preferences
- Frontend must send verified data as `overrides` payload to commit endpoint
- Green gate: commit blocked until ALL line items (orders + expenses) have green-confirmed product/category matches
- Pagination: 10 cards per page in review modal
- "Exclude" toggle per order/expense (opt-out, not opt-in)
- Base Price and OP per Unit always editable for all items, regardless of source
- All rows (including current `needsFix`) absorbed into review modal — no separate needs-fix section
- Commit result stored as JSONB for history detail viewing
- Step-by-step execution with user approval before each step

## Known Gaps
- **Validate does not accept overrides:** `POST /api/import/validate` runs only after upload (pre-review), validating stock/report-close against original CSV data. It is never re-run after review edits. The green gate only checks product/category ID matching — not stock or report-close. A user can change qty/product in review, bypass validation, and hit backend `deductStockForOrder()` runtime failures per-order. Fix: modify validate endpoint to accept optional `overrides` payload, apply them over session data, and re-invoke from review modal after edits with commit blocked until all-clear.
